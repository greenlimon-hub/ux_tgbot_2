# Руководство по использованию бота

## Что делает бот
Бот помогает:
- собирать анкеты участников;
- выдавать ссылку в чат встречи;
- публиковать карточки участников в группе;
- отправлять организаторам полные анкеты;
- запускать мини-игры для знакомства;
- принимать обращения пользователей к администраторам.

---

# 1. Где используются команды

## В личных сообщениях с ботом
Здесь участники:
- заполняют анкету;
- редактируют профиль;
- удаляют профиль;
- играют в мини-игры;
- пишут администраторам.

Организаторы в ЛС также:
- создают встречи.

## В групповом чате встречи
Здесь организаторы:
- привязывают чат к встрече;
- публикуют объявления от имени бота;
- запускают опросы;
- повторно показывают анкету участника;
- запускают мини-игры.

---

# 2. Команды в личных сообщениях

## Общие команды для всех пользователей

### `/start`
Запуск бота и вывод основного меню.

### `/help`
Показать список команд и краткую справку.

### `/events`
Показать список открытых встреч.

### `/questionnaire`
Начать заполнение анкеты.

### `/profile`
Показать сохраненную анкету.

### `/edit_profile`
Изменить анкету без полного перепрохождения.

Как работает:
- бот покажет поля анкеты;
- пользователь выберет, что именно менять;
- изменения применятся только после подтверждения.

### `/delete_profile`
Удалить профиль.

Что происходит после подтверждения:
- анкета удаляется;
- участие во встречах удаляется;
- бот пытается удалить пользователя из чатов встреч.

### `/support`
Написать администраторам о проблеме.

Шаблоны:

```text
/support У меня не открывается ссылка на чат
```

или:

```text
/support
```

После этого бот попросит описать проблему следующим сообщением.

### `/cancel`
Отменить текущий шаг:
- анкетирование;
- редактирование анкеты;
- создание обращения в поддержку;
- ответ администратора на обращение.

### `/whoami`
Показать Telegram ID пользователя.

---

## Мини-игры в личных сообщениях

### `/games`
Показать список мини-игр и вопросов.

### `/question`
Случайный вопрос для знакомства.

### `/topics`
Три темы для разговора.

### `/jeff`
Дилемма для обсуждения.

---

## Команды только для организаторов в ЛС

### `/create_demo_event`
Создать тестовую встречу.

Используется для разработки и тестов.

### `/create_event`
Создать настоящую встречу.

Шаблон:

```text
/create_event Название | Город | Место | Время | Описание
```

Пример:

```text
/create_event Встреча студентов МИЭМ | Москва | Антикафе Точка | 25 апреля, 18:30 | знакомство, разговоры и мини-игры
```

После этого бот создаст встречу и пришлет:
- название;
- город;
- место;
- время;
- описание;
- `id`;
- `slug`.

# 3. Команды в групповом чате встречи

## Базовые команды

### `/whoami`
Показать Telegram ID.

Полезно для проверки прав.

### `/bind_event`
Привязать групповой чат к встрече.

Шаблоны:

```text
/bind_event slug_встречи
```

или:

```text
/bind_event id_встречи
```

Примеры:

```text
/bind_event vstrecha-studentov-miem
```

```text
/bind_event 3
```

После привязки:
- бот понимает, что этот чат относится к конкретной встрече;
- при входе участника в чат публикует его карточку.

---

## Команды организаторов для объявлений

### `/announce`
Обычное сообщение от имени бота.

Шаблон:

```text
/announce Текст сообщения
```

Пример:

```text
/announce Всем привет! Ждем вас к 19:00 у входа.
```

---

### `/meeting`
Шаблон полного анонса встречи.

Шаблон:

```text
/meeting Когда | Где | Что будет
```

Пример:

```text
/meeting 25 апреля, 18:30 | Антикафе Точка | знакомство, разговорные игры и чай
```

---

### `/where`
Обновить место встречи.

Шаблон:

```text
/where Место | Комментарий
```

Пример:

```text
/where Антикафе Точка | вход справа от арки
```

Можно и без комментария:

```text
/where Антикафе Точка
```

---

### `/when`
Обновить время встречи.

Шаблон:

```text
/when Время | Комментарий
```

Пример:

```text
/when 19:00 | перенесли из-за брони
```

Можно и без комментария:

```text
/when 19:00
```

---

### `/important`
Важное сообщение.

Шаблон:

```text
/important Текст важного сообщения
```

Пример:

```text
/important Не забудьте зарядить телефоны и проверьте маршрут заранее
```

---

### `/poll_custom`
Создать опрос в чате.

Шаблон:

```text
/poll_custom Вопрос | Вариант 1 | Вариант 2 | Вариант 3
```

Пример:

```text
/poll_custom Кто точно будет? | Да | Скорее да | Не смогу
```

---

## Команды работы с анкетами в группе

### `/show_profile`
Повторно показать анкету участника в группе.

Есть 2 способа использования.

### Вариант 1: по username

```text
/show_profile @username
```

Пример:

```text
/show_profile @vika_miem
```

### Вариант 2: reply на сообщение участника
Ответить этой командой на сообщение участника:

```text
/show_profile
```

Бот повторно отправит публичную карточку участника.

# 4. Мини-игры в групповом чате

## `/games`
Показать список мини-игр.

## `/question`
Отправить случайный вопрос.

Под сообщением будет кнопка:
- **Следующий вопрос**

## `/topics`
Отправить 3 темы для разговора.

Под сообщением будет кнопка:
- **Следующие темы**

## `/jeff`
Отправить дилемму для очного обсуждения.

Под сообщением будет кнопка:
- **Следующая дилемма**

## `/match`
Запустить игру «Найди совпадение».

Как использовать:
1. бот отправляет тему;
2. участники отвечают **реплаем на сообщение бота**;
3. когда все ответили, кто-то нажимает кнопку:
   - **Завершить и показать ответы**
4. бот собирает ответы и показывает их;
5. затем можно нажать:
   - **Следующая тема**

### `/match_results`
Ручной сбор ответов по текущей игре.

Это запасная команда, если нужно вывести ответы без кнопки.

# 5. Как работает поддержка пользователей

## Для участника
Участник пишет:

```text
/support Текст проблемы
```

Пример:

```text
/support Я не могу зайти в чат встречи
```

Бот рассылает обращение всем администраторам.

## Для администратора
Администратор получает сообщение от бота с кнопкой:
- **Ответить участнику**

Когда один администратор нажимает кнопку:
- кнопка исчезает у всех остальных администраторов;
- этот администратор пишет ответ;
- бот отправляет ответ участнику.

Если администратор передумал отвечать:

```text
/cancel
```

Тогда:
- ответ не отправляется;
- обращение снова становится доступным другим администраторам.

# 6. Рекомендуемый сценарий работы организатора

## Перед встречей
1. В ЛС создать встречу:

```text
/create_event Название | Город | Место | Время | Описание
```

2. Добавить бота в групповой чат встречи.

3. Внутри группы привязать встречу:

```text
/bind_event slug_встречи
```

4. Проверить, что бот — администратор чата.

---

## Во время набора участников
- участники проходят `/questionnaire`;
- бот отправляет им ссылку на чат;
- после входа бот публикует карточку участника;
- организаторы получают полную анкету в ЛС.

---

## Перед началом встречи
Можно отправить:

```text
/meeting 25 апреля, 18:30 | Антикафе Точка | знакомство и мини-игры
```

или:

```text
/announce Всем привет! Начинаем собираться к 18:20
```

---

## Во время встречи
Для поддержания общения:

```text
/question
/topics
/jeff
/match
```

---

# 7. Важные замечания

## 1. Команды с `|` нужно писать строго по шаблону
Например:

```text
/meeting Когда | Где | Что будет
```

Если разделители пропущены, бот не сможет корректно разобрать команду.

## 2. Для `/match` ответы должны быть именно reply
Иначе бот их не соберет.

## 3. Для отправки поддержки админы должны сначала открыть ЛС с ботом
То есть хотя бы один раз нажать `/start`.

## 4. Для создания ссылок и удаления пользователей из чатов бот должен быть администратором
Иметь права:
- приглашать пользователей;
- удалять/банить пользователей.
