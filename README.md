# Meetup Bot — Telegram-бот для знакомств и организации встреч

Telegram-бот для офлайн-встреч, который помогает:

- собирать анкеты участников в личных сообщениях;
- управлять видимостью полей анкеты;
- выдавать ссылку на чат встречи после подтверждения анкеты;
- автоматически публиковать карточку участника в групповом чате встречи;
- отправлять полную анкету организаторам;
- публиковать организационные сообщения от имени бота;
- запускать мини-игры и вопросы для знакомства;
- принимать обращения участников к администраторам.

## Основной сценарий работы

### Для организатора
1. Создать встречу через `/create_event` в ЛС бота.
2. Добавить бота в групповой чат встречи.
3. В групповом чате выполнить `/bind_event <slug>`.
4. Использовать команды объявлений, опросов и мини-игр.

### Для участника
1. Открыть ЛС с ботом.
2. Пройти анкету через `/questionnaire`.
3. Подтвердить анкету.
4. Получить одноразовую ссылку в чат встречи.
5. После вступления в чат бот автоматически опубликует его публичную карточку.

## Установка и запуск

### 1. Создай виртуальное окружение

```bash
python -m venv .venv
```

### 2. Активируй окружение

**Windows (PowerShell):**

```bash
.venv\Scripts\Activate.ps1
```

**Windows (cmd):**

```bash
.venv\Scripts\activate
```

**macOS / Linux:**

```bash
source .venv/bin/activate
```

### 3. Установи зависимости

```bash
pip install -r requirements.txt
```

### 4. Создай `.env`

Пример:

```env
BOT_TOKEN=your_bot_token_here
DATABASE_URL=sqlite+aiosqlite:///./app.db
ADMINS=123456789,987654321
PROJECT_NAME=meetup_bot
```

### 5. Запусти бота

```bash
python -m app.main
```

## Создание встречи

### Команда

```text

```

### Пример


- название

2. Внутри чата выполни:

После этого чат будет связан со встречей.

## Анкета участника

Анкета заполняется в ЛС через `/questionnaire`.

### Что сохраняется
- имя
- возраст
- город
- вуз
- направление / факультет / специальность
- рассказ о себе
- интересы
- как проще знакомиться
- что помогает разговориться
- что делает встречу комфортной
- что может мешать
- насколько человек общительный
- какой формат общения ближе
- вероятность прихода
- что интересно на первой встрече
- ограничения по еде
- ограничения по месту / формату
- когда удобно прийти
- можно ли фотографировать
- как человек узнал о боте
- что важно знать заранее

### Видимость полей
Часть полей всегда публичны, часть можно показывать:
- всем участникам,
- только организаторам.

После подтверждения бот собирает:
- публичную карточку для группы,
- полную карточку для организаторов.

---

## Редактирование анкеты

### `/edit_profile`
Редактирование работает через **черновик**:
- старая анкета не удаляется сразу;
- можно изменить только нужные поля;
- изменения применяются только после подтверждения;
- если нажать `/cancel`, старая анкета останется без изменений.

---

## Удаление анкеты

### `/delete_profile`
После подтверждения:
- удаляются анкеты пользователя;
- удаляется участие во всех встречах;
- бот пытается удалить пользователя из всех привязанных чатов встреч.

Для удаления из чатов бот должен быть администратором этих чатов.

---

## Ссылка в чат встречи

После подтверждения анкеты бот создает **одноразовую ссылку** в чат встречи и отправляет ее участнику в ЛС.

Используется `create_chat_invite_link(..., member_limit=1)`.

### Важно
Invite link **не управляет** тем, увидит ли пользователь старые сообщения в чате. Это отдельная настройка истории чата Telegram.

---

## Автоматическая публикация анкеты в группе

Когда участник вступает в привязанный чат встречи:
- бот проверяет, есть ли у него подтвержденная анкета;
- публикует публичную карточку в группе;
- отправляет полную карточку администраторам в ЛС.

## Организационные сообщения в группе

### `/announce`
```text
/announce Всем привет! Ждем вас к 19:00 у входа.
```

### `/meeting`
```text
/meeting 25 апреля, 18:30 | Антикафе «Точка» | знакомство, разговорные игры и чай
```



```text

```

Все эти команды доступны только администраторам / организаторам.

## Поддержка участников

### Команда в ЛС
```text
/support У меня не открывается ссылка на чат
```

Или:

```text
/support
```

После этого участник отправляет проблему следующим сообщением.

### Что происходит дальше
- обращение рассылается всем администраторам в ЛС через бота;
- под обращением есть кнопка **«Ответить участнику»**;
- один из администраторов нажимает кнопку и «забирает» обращение;
- кнопка пропадает у всех остальных администраторов;
- администратор пишет ответ;
- бот пересылает ответ участнику.

Если администратор передумал, он может написать `/cancel`, и обращение снова станет доступным другим администраторам.

## Мини-игры

### `/games`
Показывает список доступных игр.

### `/question`
Отправляет случайный вопрос для знакомства.
Работает:
- в группе;
- в ЛС.

Под сообщением есть кнопка **«Следующий вопрос»**.

### `/topics`
Отправляет 3 темы для разговора.
Работает:
- в группе;
- в ЛС.

Под сообщением есть кнопка **«Следующие темы»**.

### `/jeff`
Отправляет дилемму и вопрос для обсуждения.
Работает:
- в группе;
- в ЛС.

Под сообщением есть кнопка **«Следующая дилемма»**.

### `/match`
Запускает игру «Найди совпадение».
Работает только в группе.

Механика:
1. бот отправляет тему;
2. участники отвечают **реплаем на сообщение бота**;
3. кто-то нажимает **«Завершить и показать ответы»**;
4. бот публикует ответы;
5. под результатом есть кнопка **«Следующая тема»**.

### `/match_results`
Ручной показ результатов текущей игры.

---

## Права бота в группе

Чтобы все функции работали корректно, бота лучше сделать **администратором**.

Минимально полезные права:
- приглашать пользователей (`can_invite_users`)
- удалять сообщения (желательно, чтобы скрывать служебные команды)
- банить / удалять пользователей (для `/delete_profile`)

Если бот не администратор, часть функций будет работать не полностью.

## Хранение данных

Сейчас проект использует:
- SQLite по умолчанию;
- `Base.metadata.create_all()` для создания таблиц;
- `MemoryStorage` для FSM.

### Важно
FSM хранится в памяти, поэтому:
- при перезапуске процесса активные состояния сбросятся;
- уже сохраненные ответы анкеты при этом останутся в базе.

---

## Ограничения текущей версии

### 1. Игры `/match` хранятся в памяти процесса
Если бот перезапустится, активная игра забудется.

### 2. Polling для локальной разработки
Сейчас проект запускается через polling:

```bash
python -m app.main
```

Для production лучше перейти на webhook.

### 3. Миграции Alembic пока не подключены
Схема БД создается автоматически через `create_all()`.

### 4. История чата для новых участников не управляется invite link
Это отдельная настройка Telegram-чата.

## Типичные проблемы

### `bot_token Field required`
Проверь, что `.env` лежит в корне проекта и содержит:

```env
BOT_TOKEN=...
```

### Бот не создает ссылку в чат
Проверь, что:
- бот добавлен в нужную группу;
- бот администратор;
- у него есть право приглашать пользователей.

### Участник не получает ответ администратора
Проверь, что участник уже писал боту в ЛС и не ограничил сообщения.

### Не работает удаление из чатов через `/delete_profile`
Проверь, что бот — администратор чатов и может удалять / банить пользователей.
